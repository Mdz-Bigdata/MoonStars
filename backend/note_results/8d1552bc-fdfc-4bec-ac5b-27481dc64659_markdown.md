## 1. 项目概述与核心架构 *Content-[00:00]

本项目旨在挑战“一人公司”模式，通过构建一个**AI 短剧导演智能体**，实现从剧本分析到视频合成的全流程自动化。用户仅需输入一句话，即可生成一部完整的 AI 短剧。

- **核心目标**：实现**零成本部署**（除核心 API 费用外），自动化完成短剧制作。
- **主要工具**：
    - **Coze (确有 AI)**：用于构建智能体主体。
    - **Skill**：用于定义子智能体的功能。
    - **MCP (Model Context Protocol)**：用于调用外部工具接口。
    - **火山方舟 (Volcano Engine Ark)**：提供大模型及视频生成 API。
    - **FFmpeg**：用于本地视频后期合成。
- **架构设计**：
    - **1 个主智能体 (Host Agent)**：负责任务规划与调度。
    - **6 个子智能体 (Sub-Agents)**：通过 **Skill** 方式定义，分别负责不同环节。
        1.  **剧本分析师**：提取任务、场景，生成 AI 提示词。
        2.  **素材生成师**：生成任务和图片场景素材。
        3.  **分镜设计师**：规划镜头及运镜效果。
        4.  **手针设计师 (Keyframe Designer)**：生成每个分镜的关键帧图片。
        5.  **视频生成师**：调用模型生成视频片段。
        6.  **后期合成师**：添加字幕、配音及视频合成。

## 2. 智能体工作流程详解 *Content-[02:27]

整个执行过程分为**6 个阶段**，主智能体按顺序调度子智能体完成，并在每个阶段结束后询问用户确认，支持中途修改。

1.  **剧本分析**：识别剧本中的人物和场景，保存至项目文件。
2.  **素材生成**：调用 **MCP 工具**生成对应人物和场景的参考图片。
3.  **分镜设计**：AI 设计分镜内容，保存分镜脚本文件。
4.  **关键帧生成**：根据分镜参数生成每个分镜的**首帧图片 (Keyframe)**。
5.  **视频生成**：利用关键帧图片调用视频生成模型（如 **Jimeng 1.5**）生成视频片段。
6.  **后期合成**：将所有视频片段合成为完整短剧，添加字幕和配音。

- **任务状态管理**：主智能体需维护 **Todo List**，每次工作更新任务状态和进度。
- **交互原则**：每个阶段完成后需**用户确认**，若不满意可微调剧本或任务后继续执行。

## 3. 工具配置与环境搭建 *Content-[05:30]

实现该流程需要配置具体的开发环境和 API 接口。

- **智能体创建 (Coze)**：
    - 创建主智能体（导演），定义其角色为**AI 短剧导演**。
    - 创建 6 个子智能体的 **Skill**，可通过 AI 辅助生成 Skill 的提示词和结构。
    - **Skill 提示词优化**：若格式不对，可提供完整格式示例让 AI 反推生成。
- **MCP 工具配置**：
    - **模型选择**：使用火山方舟的 **Jimeng 1.5 (即梦 1.5)** 进行视频生成（注： transcript 中提及"C 当死 1.5"即指此模型）。
    - **API 开通**：在火山方舟控制台一键开通所有模型服务，获取 **API Key**。
    - **MCP 服务器**：在 MCP 广场搜索并配置**火山引擎极工视频生成 MCP 服务器**。
    - **配置方式**：
        - **云端配置**：直接填入 API Key 生成云端函数。
        - **本地配置**：在 Coze 设置中添加 MCP 配置，替换为个人的 **API Key**。
    - **工具功能**：包含生成视频、查询任务进度等 3 个主要方法。
- **本地环境**：
    - 安装 **FFmpeg**：Mac 可通过 **Homebrew** 安装，用于视频剪辑和合成。

## 4. 实操演示：凡人修仙传大战一拳超人 *Content-[15:51]

视频展示了利用该智能体制作《凡人修仙传》韩立大战《一拳超人》的自动流程。

- **输入主题**：韩立穿越到一拳超人城市，因超市特价引发战斗。
- **剧本生成**：AI 自动生成 6 个分镜的剧本大纲（超市偶遇、贪婪攻击、见证其出、必杀神类等）。
- **素材与分镜**：
    - 生成人物（韩立、一拳超人）和场景图片。
    - 设计分镜脚本，包含人物出场、画面描述、对白情绪等。
    - **提示词语言**：建议使用**中文提示词**以提高国产模型对 IP 的理解准确度。
- **视频生成**：
    - 基于关键帧图片调用 **Jimeng 1.5** 生成 6 个视频片段。
    - 每个片段约 5 秒，总时长约 30 秒。
- **合成输出**：
    - 使用 **FFmpeg** 连接视频片段。
    - 最终生成完整短剧视频。

## 5. 存在问题与优化方案 *Content-[25:20]

第一版自动化流程存在若干问题，视频中提出了对应的优化方案。

- **主要问题**：
    1.  **IP 还原度低**：模型内部知识对《凡人修仙传》理解不准确，形象丢失。
    2.  **视频不连贯**：单图生视频导致分镜间过渡生硬，人物易变形。
    3.  **无声与口型**：生成视频为默剧，无配音且口型不对应（Jimeng 1.5 目前不支持口型同步）。
    4.  **时长控制**：视频片段时长未精确控制（预期 8 秒，实际 5 秒）。
- **优化方案**：
    1.  **引入外部搜索**：在剧本分析阶段引入联网搜索，还原 IP 设定。
    2.  **增加关键帧**：在分镜阶段生成更多**中间关键帧**，增强视频连贯性。
    3.  **提示词优化**：使用**中文提示词**，并在 Skill 中明确人物 IP 特征。
    4.  **后期处理**：
        - 合成时烧入**字幕文件**。
        - 后期添加**配音**和**音效**（可通过 MCP 工具扩展）。
        - 使用支持口型的模型（如 **HeyGen** 或 **SadTalker** 类工具）进行后期对口型处理。
    5.  **参数传递**：在调用 MCP 工具时明确传递 **Duration** 参数以控制视频时长。

## AI 总结

本视频详细演示了如何利用 **Coze 智能体**、**Skill** 和 **MCP 协议**构建一个自动化的**AI 短剧导演**。通过编排 6 个子智能体，实现了从剧本到成片的端到端流程。虽然当前版本在 IP 还原度、视频连贯性和音频同步上仍有优化空间，但通过引入外部搜索、增加关键帧控制及后期合成优化，可显著提升短剧质量。该方案为“一人公司”模式提供了低成本、高效率的 AI 创作实践路径。
